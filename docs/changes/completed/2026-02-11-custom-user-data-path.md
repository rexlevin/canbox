## 🎯 变更概述

允许用户在Canbox设置中自定义用户数据存储位置，支持浏览目录选择、数据完整迁移、重置回默认路径等功能，解决固定存储路径的限制问题。

## 🔍 问题描述

### 当前状况
Canbox的用户数据固定存储在系统默认位置：
- Windows: `~\AppData\Roaming\canbox\Users\`
- Linux: `~/.config/canbox/Users/`
- macOS: `~/Library/Application Support/canbox/Users/`

### 用户痛点
1. 无法根据用户需求选择存储位置（如更大的数据盘）
2. 无法在多系统间共享数据（如Windows和Linux双系统）
3. 系统盘空间不足时无法灵活调整
4. 备份和迁移数据困难

### 技术挑战
1. 需要修改所有路径获取相关的代码
2. 实现安全可靠的数据迁移机制
3. 处理迁移失败的回滚问题
4. 保持与现有数据的兼容性

## 💡 解决方案

### 总体方案
新增系统配置文件`config.json`存储自定义路径，修改所有路径获取函数支持自定义路径，实现完整的数据迁移机制。

### 技术选型
- 使用`config.json`存储系统级配置
- 基于现有`pathManager.js`和`storageManager.js`扩展
- 使用Electron的`dialog.showOpenDialog`选择目录

### 架构设计
```
~/.config/canbox/
├── config.json (系统配置，存储customDataRoot)
└── Users/ (用户数据目录，支持自定义位置)

使用自定义路径时：
~/.config/canbox/
└── config.json (只保存配置)

/data/myfolder/ (customDataRoot)
└── Users/ (实际用户数据)
```

## 🛠️ 实施细节

### 关键实现步骤
1. 创建`configManager.js`管理系统配置
2. 修改`pathManager.js`支持自定义数据根路径
3. 实现`UserDataPath.vue`配置界面组件
4. 实现数据迁移机制（复制+验证+回滚）
5. 添加IPC handlers处理目录选择和保存
6. 更新中英文国际化文件

### 技术要点
- 数据迁移使用完整的复制+验证机制
- 迁移失败时完全回滚，删除新目录
- `temp/`目录不迁移，仅创建空结构
- 支持恢复默认路径并迁移回默认位置

### 遇到的挑战
- 确保数据迁移的原子性（要么成功要么完全回滚）
- 处理大容量数据迁移的性能问题
- 保证迁移过程中应用的稳定性
