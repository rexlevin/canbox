## 🎯 变更概述

重构Canbox的APP列表UI，采用新的卡片式布局，增大图标尺寸，展示更多信息（平台、分类、标签、作者），功能按钮改为图标+tooltip形式，提升用户体验和视觉吸引力。

## 🔍 问题描述

### 当前状况
现有APP列表使用简单的列表布局：
1. 图标尺寸较小（58px）
2. 信息展示有限（仅名称、版本、描述）
3. 功能按钮占用空间大（文字按钮）
4. 缺乏视觉层次和美观度

### 用户痛点
1. APP信息展示不充分，用户需要点击查看详情
2. 列表式布局单调，缺乏视觉吸引力
3. 功能按钮文字占用空间，影响信息密度
4. 无法快速识别APP的平台、分类等属性

### 技术挑战
1. 设计兼容现有数据结构的卡片组件
2. 实现渐进式迁移，新旧UI并行运行
3. 保持性能不下降，支持大量APP显示
4. 确保响应式设计和跨平台兼容性

## 💡 解决方案

### 总体方案
采用渐进式迁移策略，先新增"我的APP 预览"菜单展示新UI，逐步扩展到APP仓库和APP开发界面，最终完全替代旧UI。

### 技术选型
- **Vue 3 + Element Plus** - 构建新UI组件
- **Card组件设计** - 使用圆角阴影卡片布局
- **图标按钮** - 功能按钮改为纯图标+tooltip
- **渐进式迁移** - 新旧UI并行，降低风险

### 架构设计
```
渐进式迁移策略：
阶段1: 新增"我的APP 预览"菜单，新旧UI并行 ✓
阶段2: 扩展到APP仓库和APP开发界面
阶段3: 新UI完全替代旧UI

组件复用：
AppCard.vue (可复用组件)
├── Logo区域 (72px)
├── 信息区域 (名称、描述、元信息)
└── 操作按钮区域 (图标按钮)
```

## 🛠️ 实施细节

### 第一阶段：新增"我的APP 预览"
1. 创建`AppCard.vue`卡片组件
   - 卡片高度：120-130px（原60px）
   - Logo尺寸：72px（原58px）
   - 浅灰背景 + 圆角(8px) + 阴影
   - 信息区域：名称、版本、描述、元信息

2. 元信息展示
   - **平台**：Windows/macOS/Linux SVG图标 + tooltip
   - **分类**：emoji图标（💻 development等） + tooltip
   - **标签**：展示2个，更多用...表示，悬停显示全部
   - **作者**：👤 + 作者名
   - **导入标记**：导入的APP显示小标签

3. 功能按钮改造
   - 仅显示图标，无文字
   - 水平排列在一行
   - 悬停显示tooltip（功能名称）
   - 按钮尺寸：36x36px

4. 创建`AppListNew.vue`组件
   - 使用`AppCard.vue`展示APP列表
   - 集成到CanBox.vue的新菜单项

### 技术要点
- 卡片组件设计为可复用，支持不同场景
- 使用CSS Grid或Flexbox实现响应式布局
- 图标按钮使用Element Plus的Icon组件
- 元信息使用SVG图标和emoji增强可视化

### 遇到的挑战
1. **数据适配** - 现有数据结构需要适配新UI展示
2. **性能优化** - 卡片布局比列表布局更耗性能
3. **渐进迁移** - 保持新旧UI数据同步
4. **国际化** - 新增的UI元素需要完整的中英文支持
