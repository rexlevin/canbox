## 🎯 变更概述

创建独立的日志查看器窗口，提供实时日志显示、级别过滤、日期范围筛选、搜索高亮、日志导出和清理功能，大幅提升开发和问题排查效率。

## 🔍 问题描述

### 当前状况
Canbox已完成日志统一化，所有日志通过log4js写入文件和终端输出，但缺乏实时可视化日志查看工具：
1. 开发者需要手动打开日志文件查看
2. 无法实时查看新生成的日志
3. 缺乏日志过滤、搜索和导出功能
4. 日志文件无按日期分割，历史日志查找困难

### 用户痛点
1. 调试时需要频繁切换窗口查看日志
2. 无法快速定位特定时间段的日志
3. 缺少日志导出功能进行问题反馈
4. 日志文件过大时查看困难

### 技术挑战
1. 实现主进程和渲染进程的实时日志通信
2. 处理大量日志数据的性能问题
3. 支持按日期分割和清理历史日志
4. 保证窗口状态持久化和恢复

## 💡 解决方案

### 总体方案
创建独立的日志查看器窗口，使用IPC实时获取日志，实现丰富的日志查看和管理功能，支持按日期分割日志文件。

### 技术选型
- Electron BrowserWindow创建独立窗口
- IPC通信实现实时日志传输
- log4js dateFile appender按日分割日志
- Vue 3 + Element Plus构建UI界面
- Virtual Scrolling优化大量日志显示性能

### 架构设计
```
┌─────────────────┐    IPC     ┌─────────────────┐
│   主进程        │◄──────────►│ 日志查看器窗口  │
│ - log4js配置    │            │ - 实时显示      │
│ - 内存缓存      │            │ - 过滤搜索      │
│ - 文件管理      │            │ - 导出清理      │
└─────────────────┘            └─────────────────┘
```

## 🛠️ 实施细节

### 关键实现步骤
1. 扩展logger模块支持按日分割和内存缓存
2. 创建logIpcHandler处理所有日志相关IPC请求
3. 实现logWindowManager管理独立日志窗口
4. 开发LogViewer.vue组件提供完整日志查看功能
5. 集成到系统设置和托盘菜单

### 技术要点
- MemoryAppender缓存最近1000条日志，支持实时获取
- dateFile appender按`yyyy-MM-dd`格式分割日志文件
- 自动清理超过保留天数的历史日志
- 虚拟滚动优化大量日志显示性能
- 窗口状态自动保存和恢复

### 遇到的挑战
- 实时日志传输的性能优化
- 大量日志数据的内存管理
- 多窗口间的状态同步
- 历史日志清理的可靠性
