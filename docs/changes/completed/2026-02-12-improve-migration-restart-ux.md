## 🎯 变更概述

优化数据迁移完成后的重启体验，添加5秒倒计时提示，支持立即重启按钮，检测AppImage环境并提供手动启动提示，提升用户体验的流畅度。

## 🔍 问题描述

### 当前状况
数据迁移完成后立即重启应用，用户体验突兀：
1. 用户没有缓冲时间，迁移成功提示一闪而过
2. AppImage打包的应用无法自动重启（Electron已知限制）
3. 用户不清楚需要手动启动AppImage版本

### 用户痛点
1. 迁移成功后突然重启，缺乏过渡
2. AppImage用户遇到应用没有自动重启，感到困惑
3. 缺乏明确的下一步操作指导

### 技术挑战
1. 需要检测AppImage打包环境
2. 实现前端倒计时UI与后端重启的协调
3. 保持与现有迁移流程的兼容性

## 💡 解决方案

### 总体方案
前端显示倒计时对话框，后端延迟5秒执行重启，检测AppImage环境并统一显示手动启动提示。

### 技术选型
- 使用Element Plus的Dialog组件显示倒计时
- 检测`process.env.APPIMAGE`判断AppImage环境
- 使用`setTimeout`实现延迟重启

### 架构设计
```
迁移成功 → 显示倒计时对话框 → 5秒倒计时 → 自动重启
                            ↓
                      用户点击"立即重启" → 立即重启
                            ↓
                 AppImage环境检测 → 显示手动启动提示
```
