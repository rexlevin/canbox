## 📋 总览

Canbox 作为一个轻量级应用运行时平台，本文档规划了未来的功能演进方向，分为四个主要维度：

- **📦 用户体验增强** - 提升日常使用的便捷性
- **🔧 开发体验优化** - 改善应用开发调试体验
- **🚀 系统增强** - 提升系统稳定性和可维护性
- **🎯 特色功能** - 差异化竞争的核心能力

### 日志查看器（Log Viewer）- 2026-03-14

**优先级：** ⭐⭐⭐⭐⭐ 高  
**难度：** 🟡 中等  
**状态：** ✅ 已完成

#### 功能描述
提供独立的日志查看窗口，支持实时日志查看、过滤、搜索和导出等功能，提升开发和问题排查效率。

#### 具体功能

**日志显示**
- 实时日志显示，支持自动滚动到最新日志
- 限制显示数量（最多 500 条），避免性能问题
- 智能滚动：用户查看历史日志时不自动滚动，滚动到底部时恢复自动滚动

**日志过滤**
- 按日志级别过滤（debug/info/warn/error）
- 支持多选，可同时查看多个级别的日志
- "显示全部"快速重置过滤器

**搜索功能**
- 关键词搜索，支持普通文本匹配和正则表达式
- 搜索结果高亮显示
- 清空搜索恢复原始显示

**日志导出**
- 导出为 .txt 格式（纯文本）
- 导出为 .json 格式（结构化数据）
- 支持导出全部日志或最近 N 条日志

**日期导航**
- 日期选择器，按日期查看历史日志
- 自动列出所有可用的日志文件（按日期）
- 支持加载和解压历史日志文件（.gz 压缩格式）

**日志管理**
- 清空日志文件：清空当日日志内容，不删除文件
- 清理历史日志：显示预览对话框，选择删除特定日期的日志文件
- 日志保留天数配置：1-365 天可调整

**多日志源支持**
- 应用日志（app.log）
- 监控日志（monitor.log）
- 切换日志源时自动更新可用日期列表

**窗口特性**
- 独立窗口，可在任务栏单独显示
- 可置顶选项，保持窗口在最前
- 保存和恢复窗口状态（位置、大小、置顶状态）
- 通过系统托盘菜单快速打开

**日志文件管理**
- 按日期自动分割日志文件
- 历史日志自动压缩（.gz 格式）
- 统一的日志目录路径管理
- 日志 ID 生成（时间戳 + 毫秒 + 计数器），确保唯一性

#### 技术要点

**后端实现**
- 使用 log4js 的 dateFile appender 按日期分割日志
- 实现日志 ID 生成，支持增量获取日志
- 统一的日志目录路径管理（`getLogDir()` 函数）
- IPC 处理器：`get-logs`, `get-log-files`, `export-logs`, `clear-logs`, `cleanup-old-logs`
- 支持中英文国际化

**前端实现**
- Vue 3 Composition API + Element Plus 组件库
- 轮询机制获取增量日志（1 秒间隔）
- 虚拟滚动优化性能（最多显示 500 条）
- 使用 `v-for` 的 `:key` 绑定唯一 ID 避免重复渲染
- 支持正则表达式搜索和关键词高亮

**国际化支持**
- 完整的中英文翻译
- 对话框、提示信息、按钮文本全部国际化
- 动态获取当前语言设置

#### 相关文件

- `src/components/LogViewer.vue` - 日志查看器 UI 组件
- `modules/ipc/logIpcHandler.js` - 日志相关 IPC 处理器
- `modules/main/logWindowManager.js` - 日志查看窗口管理器
- `modules/utils/logger.js` - logger 模块增强
- `locales/zh-CN.json`, `locales/en-US.json` - 国际化翻译
- `preload.js` - 暴露日志相关 API

#### 使用方式

1. **通过系统托盘打开**：点击系统托盘菜单中的"打开日志查看器"
2. **实时查看**：选择"今天"查看当日日志，自动实时更新
3. **查看历史**：选择特定日期查看历史日志
4. **过滤日志**：勾选需要的日志级别
5. **搜索日志**：输入关键词或正则表达式搜索
6. **导出日志**：点击"导出"按钮选择格式和范围
7. **管理日志**：使用"清空日志"或"清理历史日志"管理日志文件

### 17. Matrix 账号体系集成
**优先级：** ⭐⭐⭐⭐ 高  
**难度：** 🔴 复杂  
**状态：** ⬜ 暂时搁置

#### 功能描述
集成 Matrix 作为统一的身份和同步基础设施，实现跨设备数据同步、用户身份管理，并为未来 Chat/Blog 等社交功能奠定基础。

**重要设计原则：**
- Matrix 账号体系为**可选功能**
- 未登录用户可正常使用 Canbox 所有功能（包括主题切换、APP 管理、开发等）
- 登录后仅增加跨设备同步能力，不影响现有本地功能

#### 具体功能

**阶段一：Matrix 认证（预计 2-3 周）**
- 集成 matrix-js-sdk
- 实现登录方式检测（.well-known/matrix/client）
- 支持密码登录（m.login.password）
- 支持 SSO 登录（OIDC，如 Mozilla、Google、GitHub）
- 创建或加入同步 Room
- 会话持久化（token 存储）
- 账号设置 UI（Homeserver 输入、登录方式选择、登录/登出）

**阶段二：数据同步（预计 3-4 周）**
- 实现智能合并逻辑：
  - APP 列表：去重 + 版本比较
  - 配置数据：最新优先
  - Blog 帖子：全部合并（按时间戳去重）
- 离线队列：本地操作队列 + 上线批量同步
- 冲突检测与解决：
  - 最后写入优先策略（默认）
  - 提供冲突解决对话框（用户可手动干预）
  - 记录冲突历史
- 同步状态管理：
  - 同步状态指示器（idle, syncing, error）
  - 最后同步时间显示
  - 冲突通知

**阶段三：数据加密（预计 2-3 周，可选）**
- 应用层加密支持（AES-GCM）
- 主密钥管理
- 加密开关（UI）
- 密钥导入/导出功能
- 密钥恢复提示（助记词）

**阶段四：Chat 功能（预计 4-6 周，单独 Change）**
- 集成 Matrix 聊天客户端
- 私有房间管理
- 文件分享
- 富文本支持
- 消息历史

**阶段五：Blog 功能（预计 4-6 周，单独 Change）**
- ActivityPub 桥接集成
- 博客编辑器
- 博客列表展示
- 发布到 Fediverse
- 评论接收（桥接）

#### 技术要点

**认证流程设计**
```
用户输入 Homeserver URL
    ↓
获取 .well-known/matrix/client
    ↓
解析支持的登录方式
    ↓
  ┌─────────┼─────────┐
  ▼         ▼         ▼
密码登录   SSO 登录   其他方式
(m.login)  (OIDC)    (m.login.token)

用户选择并完成认证
    ↓
获取 access_token
    ↓
建立 Matrix 会话
```

**数据同步架构**
```
┌────────────┐     ┌────────────┐     ┌────────────┐
│  设备 A   │     │  设备 B   │     │  设备 C   │
└──────┬─────┘     └──────┬─────┘     └──────┬─────┘
       │                   │                   │
       │   修改             │   修改             │
       ▼                   ▼                   ▼
   ┌───────────────────────────────────────────────┐
   │         本地数据 + 离线队列                │
   └───────────────────┬───────────────────────┘
                       │
         ┌─────────────┴─────────────┐
         ▼                           ▼
   ┌─────────────────────────────────────┐
   │    智能合并逻辑               │
   │  (冲突检测与解决)              │
   └───────────────────┬─────────────┘
                       │
                       ▼
   ┌─────────────────────────────────────┐
   │    同步状态管理               │
   └───────────────────┬─────────────┘
                       │
         ┌─────────────┴─────────────┐
         ▼                           ▼
   ┌─────────────────────────────────────┐
   │     Matrix Room                │
   │  (同步数据存储)                 │
   │  - APP 列表                  │
   │  - 配置数据                   │
   │  - 用户偏好                   │
   └───────────────────┬─────────────┘
                       │
               其他设备接收同步
```

**核心模块结构**
- `modules/main/matrixAuth.js` - MatrixAuthService（认证服务）
- `modules/main/syncManager.js` - SyncManager（同步管理）
- `modules/main/encryptionManager.js` - EncryptionManager（加密管理，可选）
- `src/components/AccountSettings.vue` - 账号设置 UI
- `src/components/SyncStatus.vue` - 同步状态 UI
- 扩展 `pathManager` 支持同步数据读取

#### 风险与权衡

**风险 1：Matrix Homeserver 依赖**
- 描述：用户选择的 Homeserver 可能不稳定或不可用
- 缓解措施：允许用户更换 Homeserver、提供公共 Homeserver 列表推荐、同步失败时提示用户检查网络和 Homeserver 状态

**风险 2：同步冲突的复杂性**
- 描述：多设备同时修改数据可能产生复杂的冲突
- 缓解措施：采用最后写入优先策略、提供冲突解决对话框、记录冲突历史

**风险 3：离线队列数据丢失**
- 描述：应用崩溃可能导致未同步的离线操作丢失
- 缓解措施：持久化队列到本地存储、操作成功后才从队列移除、提供手动触发同步的选项

**权衡 1：实时性 vs 可靠性**
- 权衡：接受延迟（通常几秒内），优先保证可靠性

**权衡 2：复杂度 vs 功能完整性**
- 权衡：分阶段实现，先核心后扩展

- **设计文档：** `openspec/changes/matrix-account-system/design.md`
- **架构：** 完整的技术架构、数据流图、组件关系图
- **实施计划：** 五个阶段的详细任务列表

- **当前状态：** 暂时搁置，优先实现其他高价值功能
- **优先级调整：** 虽然是高价值功能，但实现复杂度高，建议在基础功能完善后再考虑
- **保留原因：** 完整的设计文档已准备好，未来可随时启动实施
